Dirección IP, dirección del sitio web visitado anteriormente (encabezado de la solicitud del referer), fecha y hora de la solicitud, diferencia horaria con respecto al meridiano de Greenwich, contenido de la solicitud, código de estado HTTP, volumen de datos transferidos, sitio web desde el que se realiza la solicitud e información sobre el navegador y el sistema operativo.

Aviso legal

Por regla general, se debe celebrar un contrato de procesamiento de datos con el proveedor de alojamiento. La Oficina Estatal de Baviera para la Supervisión de la Protección de Datos ha hecho una excepción para el alojamiento de sitios web puramente estáticos. En el caso de que el sitio web sirva para la presentación propia, por ejemplo, de asociaciones o pequeñas empresas, no se transfieran datos personales al operador y no se realice ningún seguimiento, no se considera que se trate de un procesamiento de datos. Además, se añade: «El hecho de que, incluso en el alojamiento de sitios web estáticos, sea inevitable el tratamiento de direcciones IP, es decir, de datos personales, no da lugar a la presunción de un tratamiento por encargo. Esto no sería adecuado. El almacenamiento (a corto plazo) de direcciones IP debe atribuirse más bien a la mediación del acceso a las telecomunicaciones por parte del proveedor de alojamiento web, de conformidad con la TKG, y sirve principalmente para fines de seguridad del proveedor de alojamiento». (https://www.lda.bayern.de/media/veroeffentlichungen/FAQ_Hosting_keine_Auftragsverarbeitung.pdf)

Partimos de la base de que esta excepción es aplicable a GitHub Pages.
